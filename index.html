Subnetting Guide (quick reference)
  1. Host bits
    Smallest power of 2 that fits hosts (+2 for network/broadcast).
    120 hosts → 2⁷ = 128 (126 usable)
  2. CIDR
    /prefix = 32 − host bits
  3. Mask
    Write the dotted-decimal mask for the prefix.
  4. Magic number
    In the subnetted octet:
    256 − maskOctet = increment
  5. List Network IDs
    Count by increment:
    .0, .32, .64, …
  6. For each subnet
    Network → usable range → broadcast
    Net | (Net+1 … Bcast−1) | Bcast
Example walkthrough (2 subnets, 110 hosts each)

Scenario: You’re assigned 192.168.100.0/24. You need two subnets that each support 110 hosts.

  1. Host bits
    Find the smallest power of 2 that can hold the required hosts (+2 reserved for network + broadcast).
    Need 110 hosts → check powers of 2: 2^6 = 64 (too small) 2^7 = 128 (works) Usable hosts = 128 − 2 = 126 (≥ 110)
  2. CIDR
    IPs are 32 bits total. CIDR = 32 − host bits.
    Host bits = 7 → CIDR = 32 − 7 = /25
  3. Subnet mask
    Write /25 in binary: 25 network 1s, then 0s. Convert to decimal.
    /25 = 11111111.11111111.11111111.10000000 Mask = 255.255.255.128
  4. Magic number
    The “interesting octet” is where subnetting happens (here: 4th octet). Magic number = 256 − that mask octet.
    Interesting octet = 4th (…255.255.255.128) Magic number = 256 − 128 = 128 (Example: 255.255.255.192 → 256 − 192 = 64)
  5. Network IDs
    Start at the first network in your assigned block, then add the magic number.
    Assigned: 192.168.100.0/24 Subnets (/25) increment by 128: Subnet 0 Network ID: 192.168.100.0 Subnet 1 Network ID: 192.168.100.128
  6. Broadcast + usable range
    Broadcast = (Network ID + Magic Number − 1). Usable range is between network and broadcast.
    Subnet 0: Net: 192.168.100.0 Bcast: 192.168.100.127 (0 + 128 − 1) Usable: 192.168.100.1 → 192.168.100.126 Subnet 1: Net: 192.168.100.128 Bcast: 192.168.100.255 (128 + 128 − 1) Usable: 192.168.100.129 → 192.168.100.254
  7. Fill the worksheet fields
    For each subnet, your tool fields map like this:
    Mask → 255.255.255.128 CIDR → /25 Network → Net Broadcast → Bcast First → Net + 1 Last → Bcast − 1
